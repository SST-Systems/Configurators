# Extensions for Config

[English](README.md) | **Русский**

Этот пример про **экстеншены (extensions)** — опциональные значения, которые ты навешиваешь на конфиг, *не трогая
его класс*. Базовый конфиг остаётся крошечным; всё дополнительное (иконка, отображаемое имя, цвет) добавляется
отдельным экстеншеном в инспекторе — только там, где реально нужно.

Здесь валюта — это просто `id` и `amount`. У одной валюты вдобавок есть иконка, имя и цвет; у другой нет вообще
ничего лишнего. Вьюшка спрашивает каждый конфиг «а есть имя? иконка? цвет?» и рисует только то, что реально
присутствует. Так одни валюты можно сделать богатыми, а другие оставить голыми — а когда позже придумаешь совсем
новый фасет, ты добавляешь тип-экстеншен, а не поле в `CurrencyConfig` и не ветку в вьюшке.

## Что внутри

- `CurrencyConfig` (ScriptableObject) — только `id`/`amount` плюс `ExtensionProcessor`, который несёт опциональные фасеты.
- Экстеншены: `DisplayName`, `Abbreviation`, `TintColor`, `Icon` (inline sync) и `IconById` (async handler,
  тянет спрайт из `SpriteRegistry`).
- `CurrencyView` — читает `TryGetExtension<T>` и отражает только присутствующие значения.
- `SpriteRegistry` — заглушка asset-менеджера; `Bootstrap` регистрирует `ExtensionManager`.

## Запуск

Открой `Scenes/Sample Scene.unity` и нажми Play. Одна строка валюты показывает иконку, имя и цвет, другая — только
сумму, при этом компонент `CurrencyView` один и тот же; всё решает лишь то, какие экстеншены несёт конкретный
конфиг. Открой ассеты валют в инспекторе и добавь/убери экстеншены — строка изменится, и это без единой правки кода.

> `TryGetExtension` возвращает первый найденный; для нескольких значений одного типа используй `GetExtensions<T>`.
> Один и тот же конфиг резолвь в одной вьюшке (повторный `ResolveExtensions` диспозит предыдущий биндинг).
